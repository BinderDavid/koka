Low level time stamps.

These are unit-less and may not necessarily be in (TAI) SI seconds, and are used internally to do time scale conversions and calendar calculations. User code should use durations and instants instead.

alias timespanstd/time/timestamp/timespan: V = ddoublestd/num/ddouble/ddouble: V

Time spans are just a ddoublestd/num/ddouble/ddouble: V; this gives it very high precision and range to represent time very precisely. (see the instant module).

type timestampstd/time/timestamp/timestamp: V

A time stamp is a timespanstd/time/timestamp/timespan: V since some epoch. Time stamps also keep track of possible leap seconds when the unit is in Unix/UTC seconds.
When using Unix/UTC seconds, the leap-adjuststd/time/timestamp/leap-adjust: (timestamp : timestamp) -> int field is non-zero if this timestampstd/time/timestamp/timestamp: V is inside a leap second. It is not the total number of leap seconds that have occurred up to that time, but is used as an adjustment to signify the time is in a particular leap second. For example, for a regular leap second we have:
2016-12-31T23:59:59Z = 536543999 with a leap-adjuststd/time/timestamp/leap-adjust: (timestamp : timestamp) -> int of 0.
2016-12-31T23:59:60Z = 536543999 with a leap-adjuststd/time/timestamp/leap-adjust: (timestamp : timestamp) -> int of 1.
2017-01-01Z = 536544000 with a leap-adjuststd/time/timestamp/leap-adjust: (timestamp : timestamp) -> int of 0 again.
